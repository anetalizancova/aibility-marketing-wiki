3. Workflow: Webináře

Každé 1–2 týdny Follow-up: [A] Vizuály + tracking: [J]

3a. Příprava před akcí

[A]

  • Připravit promo obsah — texty na social (LI, IG, FB, X)
  • Vytvořit promo emaily do Brevo (pokud je potřeba)
  • Nastavit UTM parametry pro registrační link

[J]

  • Připravit základní promo vizuály (grafiky pro sociály + event cover)
  • Spustit Meta Ads — trigger je Slack reminder z automatizace (10 a 14 dní před akcí)
  • Ověřit, že event tracking funguje

3b. Follow-up po webináři (do 24 h)

Celý proces patří [A]

  1. Transcript — přidat .md soubor do content/emails/Webinars/
  2. Vygenerovat follow-up email — Cursor command /fupwebinar → vytvoří .md draft + .html email
  3. Doplnit linky — YouTube (záznam), SoundCloud (podcast), Google Drive (materiály), feedback form
  4. UTM parametry ke všem linkům — konvence: AIB_WBN_BRO_5_RET_CZE_[NazevWebinare]
  5. Upload do Brevo — command /brevo → vytvoří draft, odešle test email
  6. Circle.so post — /circleedu (Edu space) + /circlepost (obecný space)
  7. Schválit a odeslat kampaň v Brevo (nebo naplánovat na ráno)
  • Transcript .md přidán
  • Follow-up email vygenerován (/fupwebinar)
  • YouTube / SoundCloud / Drive linky doplněny
  • UTM parametry přidány ke všem linkům
  • Upload do Brevo (/brevo)
  • Test email zkontrolován
  • Circle.so post — Edu space
  • Circle.so post — obecný space
  • Kampaň odeslána / naplánována

3c. AI Morning Show

Samostatný live stream formát. Follow-up flow je stejný jako u webinářů, jen s vlastními commands:

/AIMSfupmail — follow-up email ze záznamu/přepisu (šablona v content/emails/AI Morning Show/)
/AIMScircle — Circle post jako .md → pak /circlepost k publikaci

4. Workflow: Newsletter

Každý týden [A]
  1. Obsah — napsat v Markdownu, umístění: content/emails/NEWSLETTER/
  2. HTML verze — šablona z NEWSLETTER/ nebo Email Builder — zkontrolovat mobil, dark mode, emoji
  3. Campaign name — formát: AIB_GEN_BRO_5_RET_CZE_Newsletter[MesicRok]
  4. UTM parametry ke všem linkům
  5. Upload do Brevo — command /brevo → draft + test email
  6. Zkontrolovat v Brevo — subject, odesílatel, zobrazení, CTA linky
  7. Odeslat / naplánovat — standardně út nebo st ráno (8–9 h)
  • Obsah napsán
  • HTML šablona hotová
  • Campaign name vygenerován
  • UTM parametry na všech linkách
  • Upload do Brevo (/brevo)
  • Test email zkontrolován
  • Kampaň naplánována / odeslána

4b. AI Edu NL

Samostatný týdenní newsletter pro účastníky AI Edu Streamu (jiný seznam než hlavní Aibility newsletter). Tvorba přes command /AIEduNL — zadáš tip týdne, události a CTA, command nejdřív vygeneruje text ke schválení, pak HTML. Šablony v content/emails/AI Edu NL/.

5. Workflow: Social Media

2–3× týdně [A] LI · IG · FB · X · Threads

5a. Tvorba textu

  1. Vytvořit .md soubor v content/socials/texty/ ze šablony _TEMPLATE-social-post.md
  2. Napsat texty pro každou platformu — různý tón, délka a styl (LI: thought leadership · IG: vizuální + hashtagy · X: max 280 znaků · Threads: casual)
  3. Nastavit STATUS: ready jakmile je text hotový

5b. Vizuály

NástrojKdy použít
Gemini (/generate-image nebo standalone)Rychlé generování fotek / ilustrací
Trybloom.aiGrafiky, brandové vizuály

Ad JSON → rebrandStáhnout JSON existující reklamy → přetvořit pro Aibility brand

5c. Posting

  1. Command /upload-social [PLATFORMY] [ČAS] — např. /upload-social LI X THR nebo /upload-social IG FB zítra 10:00
  2. Zadat text postu a médium (URL obrázku nebo "bez vizuálu")
  3. Alternativa: /socialpost-auto — přečte .md soubor, naparsuje sekce, postuje přes Upload Post API
  • .md soubor vytvořen ze šablony
  • Texty napsány pro každou platformu
  • STATUS: ready nastaveno
  • Vizuál vygenerován (pokud potřeba)
  • /upload-social nebo /socialpost-auto — preview schváleno
  • Post odeslán / naplánován

6. Workflow: Email Kampaně

B2B · Launch · Sekvence [A]
  1. Obsah — napsat v Markdownu, šablona: content/emails/Aibility B2B Email Template.html
  2. HTML — šablona nebo Email Builder — zkontrolovat Outlook (VML tlačítka), mobil, dark mode
  3. Campaign name — B2B: AIB_GEN_BRO_5_CON_CZE_B2B-Osloveni · Launch: AIB_AIM_BRO_5_DEC_CZE_LaunchMar26
  4. UTM parametry na všechny linky
  5. Upload do Brevo — /brevo · pro sekvence: nastavit jako automation workflow v Brevo
  6. Segmentace — celá audience / B2B segment / webinář registrovaní
  7. Test email → schválení → odeslání

7. Workflow: Meta Ads

Průběžně Primárně [J], větší [A+J]
  1. Ověřit konverze — pokud chybí nebo nejsou spolehlivé → nejdřív nasadit tracking (viz sekce 10)
  2. Generovat copy — Cursor command meta-ads-image-headlines (headlines) + meta-ads-copy-pack (kompletní copy) — pracuje s Aibility product-bible a tone of voice
  3. Produkce vizuálů — nástroj dle potřeby:
    • Banner Ad Tool — brandované kreativy ze šablon, batch přes agenta
    • Google Whisk + Figma — generování podkladu + finalizace
    • Gemini Image Gen — AI podkladový obrázek
    • Video Ad Maker — animované video (Remotion)
    • Ad JSON → rebrand — přetvoření existující reklamy
    Export ve formátech 1×1 (feed) a 9×16 (stories)
  4. Sdílení copy (volitelné) — /mdshare → veřejný link pro odsouhlasení v týmu
  5. Založit kampaň — MetaAutomator MCP + skill meta-campaign-orchestrator
  6. Finální kontrola — konverze, kreativy, texty, budget, schedule
  • Konverze ověřeny a funkční
  • Copy pack vygenerován
  • Vizuály ve formátech 1×1 a 9×16
  • Kampaň vytvořena přes MetaAutomator
  • Finální kontrola provedena

9. Workflow: Google Ads

[J] DSA + RSA
DSA → hodně položek, které se průběžně mění (webináře, nabídky, katalog)
RSA → přesné keyword targeting, kontrolovaný messaging

Větev A: DSA

  1. Vzít vstupní URL a vytvořit skript do Google Sheetu — stahuje aktuální seznam položek
  2. Na separátním listu se generuje page feed
  3. Feed uploadnout do Google Ads
  4. Nastavit DSA kampaň nad aktuálním feedem
  5. Provozní kontrola — feed se aktualizuje, kampaň cílí na aktuální položky

Větev B: RSA

  1. Určit konkrétní landing page
  2. Keyword research — keyword research skill z Aibility marketing skills
  3. Připravit assety — headlines max 30 znaků, descriptions max 90 znaků
  4. Průběžná péče — search terms report, negativní klíčová slova

10. Workflow: Analytics & Tracking

[J] Při spouštění nové konverzní akce nebo platformy
  1. Identifikovat akci — jednoznačná, opakovatelná, byznysový smysl (příklady: assessment_started, webinar_registered, purchase)
  2. Implementace v GTM — trigger + tag + mapování parametrů + consent management
  3. Test v GTM preview/debug (povinný gate) — trigger, consent, payload, žádné duplikáty
  4. Rollout do platforem — GA4 · PostHog · Meta Pixel · Google Ads Conversion · LinkedIn Pixel (každá platforma odděleně)
  5. Verifikace — ověřit v každé platformě individuálně, event zaregistrovat/potvrdit
  6. Připravenost pro kampaně — čekat na propagaci eventu, teprve pak použít jako konverzní cíl
  • Měřitelná akce zvolena
  • GTM trigger + tag vytvořeny, consent nastaven
  • GTM preview test OK
  • Event rolloutnutý do GA4
  • Event rolloutnutý do PostHog
  • Event rolloutnutý do Meta Pixel
  • Event rolloutnutý do Google Ads Conversion
  • Event rolloutnutý do LinkedIn Pixel
  • Dostupnost pro kampaně ověřena

12. Příležitosti ke Zlepšení

Průběžně aktualizovat. Tady evidujeme bottlenecky a nápady na škálování.

Nápady ke zvážení

  • End-to-end webinář pipeline — transcript → email + Circle + social → jedno spuštění, jeden command
  • Newsletter template s dynamickými sekcemi — méně HTML ruční práce
  • Unified content calendar v Airtable — newsletter + social + webináře na jednom místě
  • Automatizovaný reporting dashboard — GA4 + Brevo + Upload Post API data
  • A/B testování subject lines — Brevo to podporuje, zatím nevyužito systematicky
  • Retargeting sekvence — web visitors → email → Meta retargeting

Kde jsme silní

  • Cursor commands výrazně zrychlují repetitivní práci
  • Content atomization — jeden webinář → email + Circle + social posty
  • UTM naming convention konzistentní — data jsou čistá
  • Vlastní tooling (Banner Ad Tool, Video Ad Maker) — žádná závislost na broken SaaS API
  • Slack reminders automatizují trigger pro webinářové ad kampaně
